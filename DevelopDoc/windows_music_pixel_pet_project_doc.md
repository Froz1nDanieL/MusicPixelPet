# Windows 音乐像素桌宠工具项目开发文档

## 0. 文档用途

本文档用于交给 AI 编程助手或开发者直接进行项目开发。项目目标是开发一个运行在 Windows 系统上的音乐像素桌宠工具。桌宠本身不内置播放器，而是监听并控制用户电脑中正在运行的国产桌面音乐播放器，第一阶段重点适配网易云音乐和 QQ 音乐。

---

## 1. 项目概述

### 1.1 项目名称

暂定名称：`Music Pixel Pet`

中文名称：`音乐像素桌宠`

### 1.2 项目目标

开发一个 Windows 桌面小工具，表现为一个透明置顶的像素风桌宠。桌宠可以：

1. 监听 Windows 系统中的媒体播放会话。
2. 只关注网易云音乐、QQ 音乐等国产桌面音乐播放器。
3. 获取当前歌曲信息，包括歌曲名、歌手、专辑、播放状态、进度、封面等可用信息。
4. 控制外部播放器，包括播放、暂停、上一首、下一首。
5. 根据当前歌曲信息和用户规则，切换桌宠的表现状态。
6. 支持鼠标交互，例如拖动桌宠、双击播放/暂停、悬停显示控制条、滚轮调节音量等。
7. 保持轻量，不做臃肿的音乐库、不内置在线音乐服务、不调用 AI 模型识别音乐风格。

### 1.3 项目定位

这是一个“音乐播放器伴侣型桌宠”，不是一个播放器。

核心价值：

- 让正在播放的音乐以桌宠动画方式反馈出来。
- 让用户可以通过桌宠快速控制外部播放器。
- 让桌宠根据音乐信息表现出不同情绪或动作。

## 2. 开发范围

### 2.1 当前版本范围

第一阶段只开发 Windows 版本。

第一阶段只监听和控制以下类型播放器：

1. 网易云音乐 Windows 桌面版。
2. QQ 音乐 Windows 桌面版。
3. 后续可扩展到酷狗音乐、酷我音乐、咪咕音乐等。

第一阶段不要求适配：

1. Spotify。
2. Apple Music。
3. 浏览器网页音乐。
4. VLC、PotPlayer、foobar2000 等通用播放器。
5. macOS、Linux。

### 2.2 明确不做的内容

MVP 不做以下内容：

1. 不内置音乐播放器。
2. 不下载音乐。
3. 不破解或绕过任何播放器限制。
4. 不读取播放器私有数据库。
5. 不调用 AI 模型识别音乐风格。
6. 不做复杂账号登录。
7. 不做歌词联网搜索。
8. 不做云同步。
9. 不做插件市场。
10. 不做复杂皮肤编辑器。

## 3. 推荐技术栈

### 3.1 总体技术栈

```text
Electron + React + TypeScript + Vite + Windows Native Helper
```

### 3.2 前端界面

使用：

```text
React
TypeScript
Vite
CSS Sprite Sheet
Zustand
```

用途：

- React：开发桌宠 UI、控制条、设置面板。
- TypeScript：降低状态和 IPC 调用出错概率。
- Vite：开发启动快，适合 Electron 前端。
- CSS Sprite Sheet：实现像素桌宠动画，学习成本最低。
- Zustand：管理前端状态，例如当前歌曲、播放状态、桌宠状态、设置项。

### 3.3 桌面容器

使用：

```text
Electron
```

用途：

- 创建透明无边框窗口。
- 创建置顶桌宠窗口。
- 托盘菜单。
- 与 Windows Helper 通信。
- 管理配置文件。
- 控制窗口位置和大小。

### 3.4 Windows 媒体监听与控制

推荐实现方式：

```text
C# .NET 8 Console Helper + Windows.Media.Control WinRT API
```

原因：

- Electron/Node 直接调用 WinRT 成本较高。
- C# 访问 Windows Runtime API 更直接。
- Helper 可以作为独立进程运行，通过标准输入输出与 Electron 通信。
- 后续如果需要替换为 Rust/N-API，也不影响前端和 Electron 主进程。

Helper 负责：

1. 获取 Windows 系统媒体会话。
2. 筛选网易云音乐、QQ 音乐等目标播放器。
3. 获取当前歌曲信息。
4. 监听播放状态变化。
5. 监听歌曲变化。
6. 执行播放、暂停、上一首、下一首等控制命令。
7. 将媒体事件通过 JSON Lines 发送给 Electron 主进程。

### 3.5 本地配置存储

使用：

```text
electron-store
```

存储内容：

- 桌宠窗口位置。
- 当前皮肤。
- 音量滚轮行为是否开启。
- 目标播放器白名单。
- 用户自定义音乐风格规则。
- 是否开机自启动。
- 是否始终置顶。
- 控制条显示方式。

### 3.6 打包工具

使用：

```text
electron-builder
```

目标输出：

- Windows `.exe` 安装包。
- 可选 portable 免安装版本。
